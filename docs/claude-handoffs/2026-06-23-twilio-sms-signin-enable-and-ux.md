## 1. PROVIDER DECISION

| | **Twilio Verify** (recommended) | **Twilio Programmable Messaging** |
|---|---|---|
| Supabase stores | Account SID, Auth Token, **Verify Service SID (`VA…`)** | Account SID, Auth Token, **Messaging Service SID** + number |
| Code generated by | Twilio | Supabase (you own the template) |
| A2P 10DLC needed for US SMS? | **No** (verification-only exempt) | **Yes** (Brand + Campaign, days of lead time) |
| Built-in fraud/retry | **Yes** (Fraud Guard) | No (we have our own rails) |
| iOS domain-bound autofill (`@host #code`) / WebOTP | Limited (Twilio owns the body) | Full |
| Time to live | **Minutes** | Days |

**Recommendation: launch on Twilio Verify** — live today, no A2P wait, free fraud protection. Trade-off: can't fully customize the SMS body, so WebOTP one-tap (Android) won't fire yet (iOS QuickType + single-field autofill still work). Migrate to Programmable Messaging later for WebOTP — **the app code does not change, only the dashboard provider.**

## 2. STEP-BY-STEP (Twilio Verify path)

**Twilio Console**
1. Confirm the account is **upgraded** (trial accounts only SMS pre-verified numbers — silent failure for real users).
2. **Verify → Services → Create** → name `Colors of Glory Auth` → copy **Service SID (`VA…`)**.
3. Copy **Account SID** + **Auth Token** from the console home.
4. Enable **Fraud Guard**; set a **billing alert** ≈ our `otp_daily_global_ceiling` (500/day).

**Supabase → Authentication → Providers → Phone**
1. **Enable** Phone provider.
2. SMS provider: **Twilio Verify**.
3. Paste **Account SID**, **Auth Token**, **Verify Service SID**.
4. OTP length **6**, expiry **600s**. Save.

**Supabase → Authentication → Rate Limits & Protection (the bypass-proof floor — required)**
An attacker can call `signInWithOtp` directly with the anon key, bypassing our app guard. Only dashboard controls stop that:
1. **Enable CAPTCHA** (Turnstile/hCaptcha). ⚠️ If enabled, the login screen must send a `captchaToken` — see §7.1; coordinate with Onboarding first or sends will fail.
2. **Allowed Countries = US/+1** (match `otp_geo_allowlist`).
3. **Provider SMS rate limit** conservative (≤30/hr to start).

## 6. THE FRICTIONLESS BUG (worth fixing)
`OTPInput.tsx` uses 6 boxes where only box 0 has `autocomplete="one-time-code"`, each `maxLength={1}`, and `handleChange` does `.slice(-1)`. When iOS Security-Code AutoFill drops the full `123456` into box 0, it's **truncated to `6`** → autofill silently fails. web.dev's fix: a single field. Paste-ready code in §7.1.

### 7.1 `src/components/cog/OTPInput.tsx` — single-field autofill (same props, same gold look)
```tsx
import { useRef, useEffect } from "react";

interface OTPInputProps {
  length?: number;
  value: string[];
  onChange: (digits: string[]) => void;
  onComplete?: (code: string) => void;
  disabled?: boolean;
  error?: boolean;
}

/** One transparent <input autocomplete="one-time-code"> under styled gold cells.
 *  iOS AutoFill, Android autofill, and full-code paste all land cleanly. */
const OTPInput = ({ length = 6, value, onChange, onComplete, disabled = false, error = false }: OTPInputProps) => {
  const inputRef = useRef<HTMLInputElement>(null);
  const code = value.join("").replace(/\D/g, "").slice(0, length);
  useEffect(() => { inputRef.current?.focus(); }, []);
  const setCode = (raw: string) => {
    const clean = raw.replace(/\D/g, "").slice(0, length);
    onChange(Array.from({ length }, (_, i) => clean[i] ?? ""));
    if (clean.length === length) onComplete?.(clean);
  };
  const focusedIndex = Math.min(code.length, length - 1);
  return (
    <div className="relative">
      <input
        ref={inputRef} type="text" inputMode="numeric" autoComplete="one-time-code"
        name="one-time-code" pattern="[0-9]*" maxLength={length} value={code} disabled={disabled}
        aria-label={`Enter the ${length}-digit code`} onChange={(e) => setCode(e.target.value)}
        className="absolute inset-0 z-10 h-full w-full opacity-0 disabled:cursor-not-allowed"
        style={{ caretColor: "transparent" }}
      />
      <div className="flex gap-2 justify-center pointer-events-none">
        {Array.from({ length }).map((_, idx) => {
          const char = code[idx] ?? "";
          const highlight = (char !== "" || (idx === focusedIndex && !disabled)) && !error;
          return (
            <div key={idx}
              className="flex items-center justify-center text-2xl font-bold rounded-2xl transition-all duration-150"
              style={{
                width: 48, height: 64, backgroundColor: "#FFFFFF",
                border: error ? "1.5px solid #E05440" : highlight ? "1.5px solid var(--cog-gold)" : "1.5px solid rgba(0,0,0,0.12)",
                color: "#1A1A1A", fontFamily: "var(--font-body)",
                boxShadow: highlight ? "0 0 0 3px var(--cog-gold-glow)" : "none",
                opacity: disabled ? 0.4 : 1,
              }}>
              {char}
            </div>
          );
        })}
      </div>
    </div>
  );
};
export default OTPInput;
```

### 7.2 `src/pages/auth/CodeVerifyPage.tsx` — WebOTP one-tap (Android), add after `handleVerify`
```tsx
// WebOTP — one-tap autofill on Android Chrome. Silent where unsupported (iOS uses
// native QuickType). Fires once the SMS is domain-bound (`@host #code`) — phase 2.
useEffect(() => {
  if (typeof window === "undefined" || !("OTPCredential" in window)) return;
  const ac = new AbortController();
  navigator.credentials
    .get({ otp: { transport: ["sms"] }, signal: ac.signal } as unknown as CredentialRequestOptions)
    .then((cred) => {
      const clean = ((cred as { code?: string } | null)?.code ?? "").replace(/\D/g, "").slice(0, CODE_LENGTH);
      if (clean.length === CODE_LENGTH) { setDigits(clean.split("")); void handleVerify(clean); }
    })
    .catch(() => { /* dismissed / unsupported / aborted */ });
  return () => ac.abort();
}, [handleVerify]);
```
