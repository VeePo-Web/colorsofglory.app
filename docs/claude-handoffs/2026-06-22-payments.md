# Payments — Claude Code Handoff (2026-06-22)

Stripe is fully enabled in Lovable Cloud (seamless integration, no BYOK key).
Everything backend + SDK is shipped. This doc is the menu for Claude Code.

## TL;DR

- Sandbox token is wired (`VITE_PAYMENTS_CLIENT_TOKEN`, prefix `pk_test_`).
- Live token auto-populates after Stripe go-live — no code change needed.
- All Stripe calls go through `@/integrations/cog/billing.ts`. Don't import
  `stripe` or `@stripe/stripe-js` directly outside `src/components/pricing/`.

## SDK — `@/integrations/cog/billing.ts`

```ts
import {
  startCheckout,            // plan-based checkout (preferred)
  createCheckoutSession,    // raw priceId (e.g. storage add-ons)
  validateCode,             // pre-checkout founder/referral code preview
  openBillingPortal,        // Stripe-hosted billing portal URL
  cancelSubscription,       // end-of-period cancel by default
  getPricingCatalog,        // plan_tiers rows
  getPricingPage,           // copy + cards for /pricing
  getLatestSubscription,    // raw row
  getEffectiveStorageLimit, // bytes (plan + add-ons)
  getMyReferralStats,
} from "@/integrations/cog/billing";
```

### Open checkout (the 90% case)

```ts
const { clientSecret } = await startCheckout({
  plan_key: "pro",
  code: enteredCode,                        // optional founder/referral code
  return_url: `${window.location.origin}/checkout/success?session_id={CHECKOUT_SESSION_ID}`,
  environment: getStripeEnvironment(),      // from "@/lib/stripe"
});
// then mount <CheckoutModal clientSecret={clientSecret} onClose={...} />
// from "@/components/pricing/CheckoutModal"
```

### Reactive subscription state

```tsx
import { useSubscription } from "@/hooks/useSubscription";

const { isPro, plan, inGracePeriod, storageLimitBytes, isLoading } = useSubscription();
if (isLoading) return <Spinner />;
if (!isPro) return <UpgradePrompt />;
```

Hook auto-refetches via realtime when `subscriptions` or `storage_addons`
rows change for the current user.

### Test mode banner

Mount once in the root layout (above main content):

```tsx
import { PaymentTestModeBanner } from "@/components/cog/PaymentTestModeBanner";

<PaymentTestModeBanner />
```

Renders nothing in live mode. Restyle if a designed treatment is wanted.

## Return URL convention

Always include `{CHECKOUT_SESSION_ID}` — Stripe replaces it server-side.

```
${origin}/checkout/success?session_id={CHECKOUT_SESSION_ID}
```

Route exists: `/checkout/success → CheckoutSuccessPage`.

## Lanes

- **Lovable** owns: edge functions, webhook, plan-gate enforcement, schema,
  `@/integrations/cog/billing.ts`, `@/lib/stripe.ts`,
  `useSubscription`, `PaymentTestModeBanner`.
- **Claude Code** owns: visual design of `/upgrade`, `/checkout/success`,
  the storage warning surface, gating treatments, copy.

If a backend behavior is missing (new plan, new gate, new edge function),
open a ticket — don't shim it in components.
